## 2026-07-23 14:41:35 -07:00 — Working-window handoff

Reviewed against current checkout commit `7707968`.

Corrections the oracle should carry into later handoffs:

- The current rewrite has no headless asset-generation contract. The private
  `IsolateSession._generation` value is GUI display lifecycle state and must not
  be placed in reusable source requests, batches, or scientific identity.
- `ActiveAsset` is immutable but shares a module with the PyQt
  `ActiveAssetController`. A genuinely headless contract must not import that
  module; use a headless request/reference plus a thin GUI snapshot adapter.
- The active asset snapshot does not currently contain the active media's
  `content_sha256`, although the asset sidecar does. Reading the recorded
  sidecar hash is not the same as verifying the current media bytes. Ordinary
  resolution may compare cheap current facts such as size, dimensions, and
  rational fps, but explicit content verification remains separate.
- Scientific source requests should require an existing registered sidecar.
  They must not silently initialize a raw video merely to manufacture stable
  identity.
- The current media API exposes only FFmpeg-produced `rgb24` bytes through
  `MediaSession.read_frame_rgb`. It has no general plane ids, plane registry,
  luma/alpha/native-plane delivery, or multi-plane batches.
- Isolate display explicitly requests a width-limited preview. Scientific
  source handoffs must require native decode and must not reuse that preview as
  evidence.
- `MediaSession.frame_count` may come from decoded, packet, container, or
  duration-derived counts. Later handoffs must retain this provenance and must
  not call every admitted extent verified.
- The current `MediaSession` owns one mutable FFmpeg cursor. A scientific
  iterator must use a request-local session rather than sharing the live player
  session.
- For a source with no consumer, the smallest bounded contract is synchronous
  iteration with natural backpressure. Do not require a worker, producer queue,
  GUI latest-only mailbox, or cross-thread cancellation until a real channel
  consumes the source.
- A source stream must make its final outcome accessible after ordinary
  iteration and must close request-local media on normal completion,
  cancellation, failure, explicit close, and early consumer exit.
- Current short RGB reads all become `FRAME_DECODE_FAILED`; the media layer does
  not preserve a structured clean-EOF versus decode-error reason. Do not require
  the working-window layer to guess truncation from message text or stderr.
- Decoded/delivered source span is not examined scientific coverage. Reserve
  `examined`, `processed`, `quiet`, and `detected` for completed scientific
  nodes.
- The current backend returns a complete exact-sized frame or an error. General
  spatial validity masks and coverage artifact types are not yet justified.
- A standalone headless diagnostic cannot observe another running Qt process's
  private current window or generation without an explicitly designed bridge.
  Until a real GUI consumer exists, use explicit diagnostic inputs and a pure
  GUI request-snapshot test.
